## 1. 环境配置问题

### Q1.1: Python 版本不兼容

**问题**: 提示 `SyntaxError` 或 `ImportError`

**解决方案**:
```bash
# 检查 Python 版本
python --version

# 需要 Python 3.10+
# 如果版本过低，请升级 Python

# 使用 conda 创建指定版本环境
conda create -n a-stock-quant python=3.10
conda activate a-stock-quant
```

### Q1.2: 依赖安装失败

**问题**: `pip install` 报错

**解决方案**:
```bash
# 1. 升级 pip
pip install --upgrade pip

# 2. 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 逐个安装（排查问题）
pip install pandas numpy matplotlib
pip install akshare

# 4. 如果是编译错误，安装构建工具
# Windows: 安装 Visual Studio Build Tools
# Linux: sudo apt-get install build-essential
# Mac: xcode-select --install
```

### Q1.3: 虚拟环境问题

**问题**: 无法激活虚拟环境

**解决方案**:
```bash
# Linux/Mac
source venv/bin/activate

# Windows (PowerShell)
venv\Scripts\Activate.ps1

# Windows (CMD)
venv\Scripts\activate.bat

# 如果仍然不行，重新创建虚拟环境
rm -rf venv
python -m venv venv
```

**问题**: 获取的数据有缺失值

**解决方案**:
```python
import pandas as pd

# 1. 检查缺失情况
print(df.isnull().sum())

# 2. 填充缺失值
# 前向填充

# 删除缺失行

# 3. 重新获取数据
# 可能是数据源问题，稍后重试
```

## 3. 回测问题

### Q3.1: 回测结果异常

**问题**: 收益率过高或过低

**解决方案**:
```python
# 1. 检查是否使用了前复权数据
# 未复权会导致价格跳空（除权除息）
df = ak.stock_zh_a_hist(symbol="000001", adjust="qfq")  # 前复权

# 2. 检查交易成本设置
# 确保包含佣金、印花税、滑点
config = {
    "commission": 0.00025,  # 万 2.5
    "stamp_tax": 0.0005,    # 0.05%
    "slippage": 0.001       # 0.1% 滑点
}

# 3. 检查是否有未来函数
# 确保只使用历史数据做决策
```

### Q3.2: 回测速度太慢

**问题**: 回测运行时间过长

**解决方案**:
```python
# 1. 减少回测时间范围
# 从 10 年减少到 3 年

# 2. 降低数据频率
# 从日线改为周线（如果策略允许）

# 3. 减少股票池
# 从全市场减少到部分股票

# 4. 使用向量化计算
# 避免使用循环，使用 pandas 向量化操作

# 5. 启用多进程
from multiprocessing import Pool
```

### Q3.3: 回测与实盘差异大

**问题**: 回测盈利，实盘亏损

**原因分析**:
1. 交易成本低估
2. 滑点未考虑
3. 流动性忽略
4. 幸存者偏差

**解决方案**:
```python
# 1. 增加交易成本
config["commission"] = 0.0003  # 提高佣金
config["slippage"] = 0.002     # 增加滑点

# 2. 考虑流动性
# 只交易成交量大于阈值的股票
df = df[df['volume'] > 1000000]

# 3. 使用完整股票池
# 包含已退市股票，避免幸存者偏差

# 4. 降低预期
# 回测收益打 5-7 折作为实盘预期
```

## 5. 交易模拟问题

### Q5.1: 无法买入股票

**问题**: `buy()` 函数返回失败

**解决方案**:
```python
# 1. 检查资金
print(f"可用资金：{account.available_cash}")
print(f"需要资金：{price * volume}")

# 2. 检查委托数量
# 必须是 100 的整数倍
assert volume % 100 == 0, "委托数量必须是 100 的整数倍"

# 3. 检查涨跌停
# 涨停时无法买入
if current_price >= limit_up_price:
    print("涨停，无法买入")

# 4. 检查交易时间
# 非交易时间无法下单
```

### Q5.2: 无法卖出股票

**问题**: `sell()` 函数返回失败

**解决方案**:
```python
# 1. 检查持仓
print(f"总持仓：{position.total_volume}")
print(f"可用持仓：{position.available_volume}")

# 2. 检查 T+1 规则
# 当日买入的份额次日才能卖出
if buy_date == trade_date:
    print("T+1 限制：当日买入不可卖出")

# 3. 检查涨跌停
# 跌停时无法卖出
if current_price <= limit_down_price:
    print("跌停，无法卖出")
```

### Q5.3: 成交与预期不符

**问题**: 委托未成交或成交价格不符

**解决方案**:
```python
# 1. 检查委托类型
# 限价委托：价格不优可能不成交
# 市价委托：保证成交但价格不确定

# 2. 检查撮合逻辑
# 买入：委托价 >= 卖一价才能成交
# 卖出：委托价 <= 买一价才能成交

# 3. 调整委托价格
# 买入时提高委托价
# 卖出时降低委托价

# 4. 使用市价单
order_type = "market"  # 市价委托
```

## 6. 其他问题

**解决方案**:

**解决方案**:

### Q6.3: 内存不足

**问题**: `MemoryError`

**解决方案**:
```python
# 1. 减少数据量
# 只加载需要的列
df = pd.read_csv('data.csv', usecols=['date', 'close', 'volume'])

# 2. 分块处理
for chunk in pd.read_csv('data.csv', chunksize=10000):
    process(chunk)

# 3. 删除不需要的变量
del large_object
import gc
gc.collect()

# 4. 使用更高效的数据类型
df = df.astype({'col1': 'int32', 'col2': 'float32'})
```
