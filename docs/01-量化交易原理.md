## 1. 什么是量化交易

### 1.1 定义

**量化交易 (Quantitative Trading)** 是指利用数学模型、统计方法和计算机技术，通过分析市场数据来制定交易决策，并由程序自动执行交易的一种投资方式。

### 1.2 量化交易 vs 主观交易

| 维度 | 量化交易 | 主观交易 |
|------|---------|---------|
| 决策依据 | 数据、模型、规则 | 经验、直觉、判断 |
| 执行方式 | 程序自动执行 | 人工手动执行 |
| 情绪影响 | 无 | 大 |
| 可复制性 | 高 | 低 |
| 处理能力 | 可同时处理大量标的 | 有限 |
| 反应速度 | 毫秒级 | 秒级或更慢 |

### 1.3 量化交易的优势

```
┌─────────────────────────────────────────────────────┐
│  纪律性  │  严格执行策略，克服人性弱点              │
│  系统性 │  多维度分析，全面覆盖市场                │
│  及时性  │  快速响应市场变化                        │
│  准确性  │  精确计算，减少人为错误                  │
│  可验证  │  策略可回测，效果可量化                  │
└─────────────────────────────────────────────────────┘
```

## 2. 量化交易的核心流程

### 2.1 完整流程图

```
┌──────────────────────────────────────────────────────────────────┐
│                       量化交易完整流程                            │
└──────────────────────────────────────────────────────────────────┘

  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
  │ 数据收集 │ →  │ 策略研究 │ →  │ 回测验证 │ →  │ 模拟交易 │ →  │ 实盘交易 │
  └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
       ↓              ↓              ↓              ↓              ↓
  历史数据      信号生成       绩效评估       风险控制       实时监控
  实时数据      参数优化       风险分析       压力测试       动态调整
  另类数据      组合构建       过拟合检验     资金管理       日志记录
```

### 2.2 各环节详解

#### 2.2.1 数据收集

**数据类型：**

| 数据类型 | 内容 | 来源 |
|---------|------|------|
| 行情数据 | 开盘价、收盘价、最高价、最低价、成交量 | 交易所、数据商 |
| 财务数据 | 营收、利润、资产负债、现金流 | 财报、数据商 |
| 宏观数据 | GDP、CPI、利率、汇率 | 统计局、央行 |
| 另类数据 | 新闻舆情、社交媒体、卫星图像 | 网络爬虫、API |

**数据质量要求：**
- ✅ 准确性：数据无误
- ✅ 完整性：无缺失
- ✅ 一致性：格式统一
- ✅ 及时性：更新及时

#### 2.2.2 策略研究

**研究步骤：**

1. **假设提出** - 基于市场观察或理论提出交易假设
2. **数据探索** - 分析数据特征，寻找规律
3. **模型构建** - 建立数学模型或统计模型
4. **信号生成** - 定义买入/卖出条件
5. **参数优化** - 寻找最优参数组合

#### 2.2.3 回测验证

**回测核心要素：**

```python
# 回测需要模拟的真实交易要素
回测 = {
    "交易成本": ["佣金", "印花税", "过户费", "滑点"],
    "交易规则": ["T+1", "涨跌停", "最小交易单位"],
    "数据质量": ["前复权", "后复权", "停牌处理"],
    "评估指标": ["收益率", "夏普比率", "最大回撤"]
}
```

**常见回测陷阱：**

| 陷阱 | 说明 | 避免方法 |
|------|------|---------|
| 未来函数 | 使用了未来数据 | 严格检查数据时间点 |
| 过拟合 | 参数过度优化 | 样本外测试、交叉验证 |
| 幸存者偏差 | 只考虑存活股票 | 使用完整历史数据 |
| 流动性忽略 | 忽略成交难度 | 考虑成交量限制 |
| 成本低估 | 低估交易成本 | 完整计算所有费用 |

#### 2.2.4 模拟交易

**目的：**
- 验证策略在实时市场的表现
- 测试交易系统稳定性
- 熟悉交易流程
- 建立交易信心

**与回测的区别：**

| 维度 | 回测 | 模拟交易 |
|------|------|---------|
| 数据 | 历史数据 | 实时数据 |
| 成交 | 理想化 | 接近真实 |
| 速度 | 快速 | 实时 |
| 心理 | 无压力 | 有压力 |

#### 2.2.5 实盘交易

**实盘准备清单：**

- [ ] 策略经过充分回测和模拟
- [ ] 建立风险管理制度
- [ ] 准备应急预案
- [ ] 设置监控告警
- [ ] 准备充足资金
- [ ] 选择可靠券商和通道

## 3. 有效市场假说

### 3.1 理论概述

**有效市场假说 (Efficient Market Hypothesis, EMH)** 由 Eugene Fama 于 1970 年提出，认为市场价格已经反映了所有可获得的信息。

### 3.2 三种有效形式

```
┌─────────────────────────────────────────────────────────────┐
│                     市场有效性光谱                           │
│                                                             │
│  弱式有效  ←────────→  半强式有效  ←────────→  强式有效     │
│                                                             │
│  历史信息   所有公开信息   所有信息（包括内幕）              │
│                                                             │
│  技术分析无效  基本面分析无效  任何分析都无效                │
└─────────────────────────────────────────────────────────────┘
```

| 形式 | 价格反映信息 | 无效的分析方法 | 可能有效的方法 |
|------|-------------|---------------|---------------|
| **弱式有效** | 历史价格、成交量 | 技术分析 | 基本面分析、内幕信息 |
| **半强式有效** | 所有公开信息 | 技术分析、基本面分析 | 内幕信息 |
| **强式有效** | 所有信息 | 所有方法 | 无 |

### 3.3 A 股市场的效率

**研究结论：**

- A 股市场处于 **弱式有效** 阶段
- 技术分析仍有一定作用
- 基本面分析有效性强
- 存在行为金融学可解释的异象

**量化策略空间：**

```
由于 A 股效率相对较低，以下策略有机会获得超额收益：

1. ✅ 技术分析类策略（趋势、均值回归）
2. ✅ 基本面量化（多因子选股）
3. ✅ 行为金融策略（利用投资者非理性）
4. ✅ 事件驱动策略
```

## 4. Alpha 与 Beta

### 4.1 基本概念

```
总收益 = Beta 收益 + Alpha 收益

Beta (β)  = 市场收益，跟随大盘涨跌
Alpha (α) = 超额收益，独立于市场的收益
```

### 4.2 Beta 收益

**特点：**
- 被动收益
- 与市场高度相关
- 容易获得
- 成本较低

**获取方式：**
- 买入并持有指数基金
- 买入一篮子股票复制指数

### 4.3 Alpha 收益

**特点：**
- 主动收益
- 独立于市场
- 难以获得
- 需要技能

**来源：**
| 来源 | 说明 |
|------|------|
| 选股能力 | 选出表现优于市场的股票 |
| 择时能力 | 在市场上涨前买入，下跌前卖出 |
| 套利机会 | 利用市场价格差异 |
| 信息优势 | 拥有更快或更准确的信息 |

### 4.4 Alpha 与 Beta 分离

**市场中性策略：**

```
目标：只获取 Alpha，对冲掉 Beta

方法：
1. 做多一篮子股票（获取 Alpha + Beta）
2. 做空股指期货（对冲 Beta）
3. 剩余收益 = Alpha
```

## 5. 风险与收益指标

### 5.1 收益指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **收益率** | (期末值 - 期初值) / 期初值 | 基础收益指标 |
| **年化收益率** | (1 + 总收益率)^(252/n) - 1 | 标准化年收益 |
| **超额收益** | 策略收益 - 基准收益 | 相对基准的表现 |

### 5.2 风险指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **波动率** | Std(日收益率) × √252 | 收益的波动程度 |
| **最大回撤** | max(峰值 - 谷值) / 峰值 | 最大亏损幅度 |
| **VaR** | Value at Risk | 一定置信度下的最大损失 |

### 5.3 风险调整收益指标

#### 夏普比率 (Sharpe Ratio)

```
夏普比率 = (策略年化收益 - 无风险利率) / 策略年化波动率

解读：
- < 0: 策略亏损
- 0-1: 收益不足以补偿风险
- 1-2: 合格
- 2-3: 优秀
- > 3: 非常优秀
```

#### 索提诺比率 (Sortino Ratio)

```
索提诺比率 = (策略年化收益 - 无风险利率) / 下行波动率

与夏普比率的区别：只考虑下行波动（坏的风险）
```

#### 卡玛比率 (Calmar Ratio)

```
卡玛比率 = 策略年化收益 / 最大回撤

解读：单位回撤带来的收益
- > 1: 合格
- > 2: 优秀
- > 3: 非常优秀
```

### 5.4 其他重要指标

| 指标 | 说明 |
|------|------|
| **胜率** | 盈利交易次数 / 总交易次数 |
| **盈亏比** | 平均盈利 / 平均亏损 |
| **交易次数** | 总交易数量 |
| **持仓周期** | 平均持仓时间 |
| **信息比率** | 超额收益 / 跟踪误差 |

## 6. A 股市场特点

### 6.1 交易规则

| 规则 | 说明 |
|------|------|
| **交易时间** | 周一至周五 9:30-11:30, 13:00-15:00（法定节假日除外） |
| **T+1 制度** | 当日买入，次日才能卖出 |
| **涨跌停板** | 主板±10%，科创板/创业板±20%，ST 股±5% |
| **最小单位** | 1 手 = 100 股，买入必须是 100 股的整数倍 |
| **印花税** | 卖出时收取成交金额的 0.05%（2023 年 8 月后） |
| **佣金** | 买卖双向收取，一般万分之 2.5 左右，最低 5 元 |

## 7. 量化交易的优势与局限

### 7.1 优势

```
┌──────────────────────────────────────────────────────┐
│  1. 纪律性   │  严格执行策略，不受情绪干扰            │
│  2. 系统性   │  全面分析，不遗漏重要信息              │
│  3. 快速性   │  毫秒级响应市场变化                    │
│  4. 精确性   │  精确计算，减少人为错误                │
│  5. 可验证   │  策略可回测，效果可量化                │
│  6. 可扩展   │  可同时交易多个标的                    │
│  7. 可复制   │  策略可标准化复制                      │
└──────────────────────────────────────────────────────┘
```

### 7.2 局限

| 局限 | 说明 | 应对方法 |
|------|------|---------|
| **模型风险** | 模型可能失效 | 多策略组合、持续迭代 |
| **历史依赖** | 基于历史数据 | 样本外测试、压力测试 |
| **黑天鹅** | 极端事件难以预测 | 严格风控、止损机制 |
| **技术门槛** | 需要编程和数学能力 | 学习提升、团队合作 |
| **数据质量** | 数据可能有问题 | 数据清洗、交叉验证 |
| **过度拟合** | 策略可能过拟合 | 简化模型、正则化 |

### 7.3 成功要素

```
成功的量化交易 = 正确的理念 + 有效的策略 + 严格的执行 + 完善的风控

1. 理念：理解市场、认识自己
2. 策略：正期望值、可执行
3. 执行：纪律严明、不打折扣
4. 风控：仓位管理、止损止盈
```
