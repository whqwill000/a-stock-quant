## 2. 代码规范

### 2.1 编码风格

遵循 **PEP 8** Python 编码规范。

**关键规则**：

```python
# 1. 缩进：使用 4 个空格，不使用 Tab
def my_function():
    if True:
        print("正确缩进")

# 2. 行宽：最大 100 字符
long_string = "这是一行非常长的字符串，超过了推荐的行宽限制，需要换行"

# 3. 空行：顶级定义之间空两行，方法之间空一行
class MyClass:
    def method1(self):
        pass
    
    def method2(self):
        pass


def standalone_function():
    pass

# 4. 导入顺序：标准库 → 第三方库 → 本地模块
import os
import sys

import numpy as np
import pandas as pd

from . import local_module
```

### 2.2 命名规范

```python
# 变量和函数：小写 + 下划线
user_name = "张三"
def calculate_return():
    pass

# 类名：大驼峰
class TradingStrategy:
    pass

# 常量：全大写 + 下划线
MAX_POSITION_RATIO = 0.30
DEFAULT_INITIAL_CASH = 1000000

# 私有成员：单下划线前缀
class Account:
    def _internal_method(self):
        pass
    
    _private_attr = None

# 避免名称冲突：双下划线前后
class MyClass:
    def __init__(self):
        self.__private = None  # 名称修饰
```

### 2.3 类型注解

所有公共 API 必须使用类型注解：

```python
from typing import List, Dict, Optional, Tuple, Union

# 函数类型注解
def calculate_profit(
    buy_price: float,
    sell_price: float,
    volume: int,
    commission_rate: float = 0.00025
) -> float:
    """计算盈利"""
    return (sell_price - buy_price) * volume * (1 - commission_rate * 2)

# 变量类型注解
positions: Dict[str, Position] = {}
orders: List[Order] = []

# 可选类型
def get_position(stock_code: str) -> Optional[Position]:
    """获取持仓，不存在返回 None"""
    pass

# 联合类型
def parse_value(value: Union[str, int, float]) -> float:
    """解析数值"""
    pass

# 泛型
class DataContainer(Generic[T]):
    def __init__(self, data: List[T]):
        self.data = data
```

### 2.4 文档字符串

使用 Google 风格的文档字符串：

```python
def buy(
    stock_code: str,
    price: float,
    volume: int,
    order_type: str = "limit"
) -> str:
    """
    买入股票
    
    Args:
        stock_code: 股票代码，如 "000001.SZ"
        price: 委托价格
        volume: 委托数量（100 的整数倍）
        order_type: 订单类型，可选 "limit" 或 "market"，默认 "limit"
        
    Returns:
        订单 ID，如果失败返回 None
        
    Raises:
        ValueError: 当委托数量不是 100 的整数倍时
        InsufficientFundsError: 当资金不足时
        
    Example:
        >>> order_id = buy("000001.SZ", 10.50, 1000)
        >>> print(f"订单 ID: {order_id}")
    """
    pass


class TradingStrategy:
    """
    交易策略基类
    
    所有策略类都应继承此类，并实现 generate_signals 方法
    
    Attributes:
        name: 策略名称
        params: 策略参数
        is_active: 策略是否激活
        
    Example:
        >>> strategy = MyStrategy(params={"window": 20})
        >>> signals = strategy.generate_signals(data)
    """
    
    def __init__(self, name: str, params: dict = None):
        """
        初始化策略
        
        Args:
            name: 策略名称
            params: 策略参数字典
        """
        self.name = name
        self.params = params or {}
```

### 2.5 异常处理

```python
# 自定义异常
class TradingError(Exception):
    """交易相关异常基类"""
    pass


class InsufficientFundsError(TradingError):
    """资金不足异常"""
    pass


class InsufficientPositionError(TradingError):
    """持仓不足异常"""
    pass


# 异常处理最佳实践
def execute_trade(order: Order) -> Trade:
    """执行交易"""
    try:
        # 检查资金
        if not account.has_enough_funds(order):
            raise InsufficientFundsError(f"资金不足：需要{order.amount}")
        
        # 检查持仓
        if not account.has_enough_position(order):
            raise InsufficientPositionError(f"持仓不足：需要{order.volume}")
        
        # 执行交易
        trade = matching_engine.match(order)
        return trade
        
    except InsufficientFundsError as e:
        logger.warning(f"交易失败：{e}")
        raise
    except Exception as e:
        logger.error(f"未知错误：{e}")
        raise TradingError(f"交易执行失败：{e}")
```

## 4. Git 工作流

### 4.1 分支模型

```
main (受保护)
  │
  ├── develop (开发分支)
  │     │
  │     ├── feature/xxx (特性分支)
  │     ├── bugfix/xxx  (修复分支)
  │     └── hotfix/xxx  (紧急修复)
```

### 4.2 分支命名

| 分支类型 | 命名格式 | 说明 |
|---------|---------|------|
| 主分支 | `main` | 生产环境代码 |
| 开发分支 | `develop` | 开发集成分支 |
| 特性分支 | `feature/xxx` | 新功能开发 |
| 修复分支 | `bugfix/xxx` | Bug 修复 |
| 紧急修复 | `hotfix/xxx` | 生产环境紧急修复 |

### 4.3 提交规范

遵循 Conventional Commits 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具/配置

**示例**：

```bash
# 新功能
git commit -m "feat(simulator): 添加 T+1 交易规则支持"

# Bug 修复
git commit -m "fix(backtest): 修复分红除权计算错误"

# 文档更新
git commit -m "docs: 更新策略详解文档"

# 重构
git commit -m "refactor(strategy): 重构多因子策略代码结构"
```

### 4.4 开发流程

```bash
# 1. 从 develop 创建特性分支
git checkout develop
git pull origin develop
git checkout -b feature/new-strategy

# 2. 开发并提交
git add .
git commit -m "feat(strategy): 实现趋势跟踪策略"

# 3. 推送分支
git push origin feature/new-strategy

# 4. 创建 Pull Request
# 在 GitHub/GitLab 上创建 PR，请求合并到 develop

# 5. Code Review
# 等待团队成员审查

# 6. 合并
# 审查通过后合并到 develop
```

### 5.3 测试用例编写

```python
# tests/test_simulator.py
import pytest
from core.simulator import TradingSimulator, Order, Position


@pytest.fixture
def simulator():
    """创建模拟器 fixture"""
    return TradingSimulator(initial_cash=1000000)


class TestTradingSimulator:
    """模拟器测试类"""
    
    def test_buy_success(self, simulator):
        """测试买入成功"""
        order_id = simulator.buy("000001.SZ", 10.50, 1000)
        assert order_id is not None
        
        order = simulator.get_order(order_id)
        assert order.stock_code == "000001.SZ"
        assert order.volume == 1000
        
    def test_buy_insufficient_funds(self, simulator):
        """测试资金不足"""
        with pytest.raises(InsufficientFundsError):
            simulator.buy("000001.SZ", 10.50, 10000000)
            
    def test_sell_success(self, simulator):
        """测试卖出成功"""
        # 先买入
        simulator.buy("000001.SZ", 10.50, 1000)
        
        # T+1: 需要过一天才能卖出
        simulator.next_day()
        
        # 卖出
        order_id = simulator.sell("000001.SZ", 11.00, 500)
        assert order_id is not None
        
    def test_t_plus1_rule(self, simulator):
        """测试 T+1 规则"""
        # 买入
        simulator.buy("000001.SZ", 10.50, 1000)
        
        # 当日不能卖出
        with pytest.raises(InsufficientPositionError):
            simulator.sell("000001.SZ", 11.00, 500)
```

## 7. 版本发布

### 7.1 版本号规范

遵循语义化版本 (Semantic Versioning)：

```
主版本号。次版本号.修订号
MAJOR.MINOR.PATCH

例如：1.2.3

- MAJOR: 不兼容的变更
- MINOR: 向后兼容的功能增加
- PATCH: 向后兼容的问题修复
```

### 7.2 发布流程

```bash
# 1. 更新版本号
# 在 core/__init__.py 中更新 __version__

# 2. 更新 CHANGELOG.md
# 记录本次版本的变更

# 3. 创建发布分支
git checkout -b release/v1.2.3

# 4. 运行测试
pytest

# 5. 构建
python setup.py sdist bdist_wheel

# 6. 创建 Git 标签
git tag -a v1.2.3 -m "Release version 1.2.3"

# 7. 推送标签
git push origin v1.2.3

# 8. 发布到 PyPI（可选）
twine upload dist/*
```
