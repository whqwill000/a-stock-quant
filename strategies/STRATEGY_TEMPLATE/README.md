## 配置说明

### 策略参数

编辑 `config.yaml` 文件：

```yaml
strategy:
  name: "策略名称"
  
  # 核心参数
  param1: value1
  param2: value2
  
  # 交易参数
  position_size: 0.1    # 单次交易仓位（10%）
  stop_loss: 0.05       # 止损比例（5%）
  take_profit: 0.15     # 止盈比例（15%）
  
  # 回测参数
  initial_cash: 1000000  # 初始资金
  commission: 0.00025    # 佣金费率
  slippage: 0.001        # 滑点
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| param1 | value1 | 参数说明 |
| param2 | value2 | 参数说明 |
| position_size | 0.1 | 单次交易仓位比例 |
| stop_loss | 0.05 | 止损比例 |
| take_profit | 0.15 | 止盈比例 |


```bash
# 编辑配置文件

```

### 3. 运行回测

```bash
# 运行回测脚本

# 或指定时间范围
python backtest.py --start 2020-01-01 --end 2023-12-31
```

## 风险提示

⚠️ **重要风险提示**：

1. **历史业绩不代表未来** - 回测结果基于历史数据，未来表现可能不同

2. **过拟合风险** - 策略可能在历史数据上过拟合

3. **市场变化** - 市场结构变化可能导致策略失效

4. **交易成本** - 实际交易成本可能高于回测假设

5. **流动性风险** - 实际交易可能面临流动性不足

6. **技术风险** - 系统故障、网络问题等可能影响交易

### 适用人群

- ✅ 有一定量化基础的投资者
- ✅ 能够承受 XX% 以上回撤
- ✅ 有充足时间监控策略运行
- ❌ 不适合风险厌恶型投资者
- ❌ 不适合期望保本保收益的投资者

### 使用建议

1. **小额试水** - 先用小资金验证策略有效性
2. **持续监控** - 定期检查策略表现
3. **及时止损** - 策略失效时及时停止
4. **分散投资** - 不要将所有资金投入单一策略
